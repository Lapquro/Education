## Задание

Написать программу на языке Си для построения и обработки **двоичного дерева поиска (BST)**. Программа работает через текстовое меню и поддерживает:

- Добавление нового узла (порядок BST сохраняется)
- Текстовую визуализацию дерева (с отступом, пропорциональным глубине)
- Удаление узла (целостность и порядок дерева сохраняются)

## Задание №6 — Нетерминальная вершина с максимальной глубиной

> **Нетерминальная вершина** — вершина, имеющая хотя бы одного потомка (то есть не лист).  
> Среди всех таких вершин найти ту, что расположена на **наибольшей глубине**, и вернуть её значение.

### Как работает алгоритм

Функция `find_deepest_nonterminal` обходит всё дерево рекурсивно, отслеживая глубину каждого узла. Когда встречается нетерминальная вершина глубже текущего лучшего результата — результат обновляется.

```
        FART_BOSS              <- глубина 0, нетерминальная
       /         \
  FART_AUDIBLE  FART_DIMENSIONAL  <- глубина 1, нетерминальные
  /        \        /
FART_SILENT FART_STINKY FART_ELEVATOR  <- глубина 2, нетерминальные
    /               \         \
STILL_ALIVE      FART_WET   FART_DATE   <- глубина 3
                               \
                           FART_FUNERAL  <- глубина 4 (лист)
```

В этом дереве ответ — **`FART_DATE`**: находится на глубине 3 и имеет потомка (`FART_FUNERAL`), что делает её самой глубокой нетерминальной вершиной.

## Два варианта

В проекте два варианта по типу данных. Алгоритм одинаковый — отличается только тип хранимого значения.

### Вариант A — `int`

Хранит обычные целые числа.

```c
typedef struct Node {
    int          data;
    struct Node *left;
    struct Node *right;
} Node;
```

Тестовое дерево: `50, 30, 70, 20, 40, 60, 80, 10, 35, 65`

```
        80
    70
            65
        60
50
        40
            35
    30
        20
            10
```

Результат: самая глубокая нетерминальная вершина = **`20`** (глубина 2, имеет потомка `10`)

---

### Вариант B — `FartLevel` (перечисление)

Хранит значения пользовательского типа `enum`. Перечисление упорядочено, поэтому сравнения BST работают корректно — константы enum под капотом являются целыми числами.

```c
typedef enum {
    STILL_ALIVE,        // 0
    FART_SILENT,        // 1
    FART_AUDIBLE,       // 2
    FART_STINKY,        // 3
    FART_WET,           // 4
    FART_BOSS,          // 5
    FART_ELEVATOR,      // 6
    FART_DATE,          // 7
    FART_FUNERAL,       // 8
    FART_DIMENSIONAL    // 9
} FartLevel;
```

Тестовое дерево: `FART_BOSS, FART_AUDIBLE, FART_DIMENSIONAL, FART_SILENT, FART_STINKY, FART_ELEVATOR, STILL_ALIVE, FART_WET, FART_DATE, FART_FUNERAL`

```
    FART_DIMENSIONAL
                FART_FUNERAL
            FART_DATE
        FART_ELEVATOR
FART_BOSS
            FART_WET
        FART_STINKY
    FART_AUDIBLE
        FART_SILENT
            STILL_ALIVE
```

Результат: самая глубокая нетерминальная вершина = **`FART_DATE`** (глубина 3, имеет потомка `FART_FUNERAL`)

## Основные функции

| Функция | Описание |
|---|---|
| `bst_insert` | Вставка значения с сохранением порядка BST |
| `bst_delete` | Удаление узла, обрабатывает все 4 случая |
| `tree_print` | Печать дерева боком с отступом по глубине |
| `find_deepest_nonterminal` | Рекурсивный обход с отслеживанием лучшей нетерминальной вершины |
| `deepest_nonterminal_value` | Публичная обёртка, возвращает 1 если нашли, 0 если дерево из одного узла |
| `fart_name` | *(только FartLevel)* Преобразует значение enum в строку с его именем |

### Удаление узла: 4 случая

```
Случай 1: нет потомков (лист)     -> освобождаем и возвращаем NULL
Случай 2: только правый потомок   -> освобождаем узел, возвращаем правого потомка
Случай 3: только левый потомок    -> освобождаем узел, возвращаем левого потомка
Случай 4: два потомка             -> заменяем значение in-order преемником,
                                     удаляем преемника из правого поддерева
```

## Определения (из условия задания)

| Термин | Определение |
|---|---|
| **Глубина вершины** | Длина пути от корня до данной вершины |
| **Глубина (высота) дерева** | Максимальная глубина среди всех вершин |
| **Лист / терминальная вершина** | Вершина, не имеющая потомков |
| **Нетерминальная вершина** | Вершина, имеющая хотя бы одного потомка |
| **Степень вершины** | Число потомков данной вершины |
| **BST** | Двоичное дерево, где левое поддерево < корень < правое поддерево |
